#+TITLE: org-skim

Emacs package that integrates the [[https://skim-app.sourceforge.io/][Skim]] PDF reader with Org mode via AppleScript.  Extract a PDF's table of contents as an Org heading tree to insert or yank, drive Skim's page navigation from Emacs, manage Skim bookmarks (including folder bookmarks), control its reading bar line by line, and link Skim annotations to Org headings you can jump between in either direction.

中文文档请见 [[file:README.zh.org][README.zh.org]]。

* Requirements

- macOS with [[https://skim-app.sourceforge.io/][Skim]] installed (the package drives it via AppleScript).
- Emacs 26.1 or newer.  A GUI build (with ~do-applescript~) is used automatically when available; a terminal build falls back to shelling out to ~osascript~.

* Installation

** straight.el / use-package

#+begin_src emacs-lisp
  (use-package org-skim
    :straight (org-skim :type git :host github :repo "smartepsh/org-skim")
    :config
    (setq org-skim-capture-in-new-frame t
          org-skim-view-note-in-new-frame t
          org-skim-page-link-format "${title} P.${page}"
          org-skim-bookmark-name-format "P.${page}"
          org-skim-open-org-note-function 'vulpea-visit ;; I'm using vulpea for note taking
          org-skim-note-title-template "Notes on ${title}/${year}"
          org-skim-note-filetags "ReadingNote"
          org-skim-note-references-property "REFERENCES"))
#+end_src

** package-vc (built-in, Emacs 29+)

#+begin_src emacs-lisp
(package-vc-install "https://github.com/smartepsh/org-skim")
#+end_src

Or declaratively with ~use-package~ on Emacs 30+:

#+begin_src emacs-lisp
(use-package org-skim
  :vc (:url "https://github.com/smartepsh/org-skim" :rev :newest))
#+end_src

* Usage

Common workflows and the functions that drive them.  Open a PDF in Skim first; all commands require a front document.

** Reading with a bookmark trail

Mark pages you want to return to, then jump back directly from Emacs.

- ~M-x org-skim-bookmark-add~ — bookmark the current page.
- ~M-x org-skim-goto-bookmark~ — pick a bookmark from completion and jump to it.
- ~M-x org-skim-bookmark-remove~ — remove a bookmark you no longer need.

** Inserting a TOC into Org

Extract the front document's outline as an Org heading tree and drop it into a note.

- ~M-x org-skim-insert-toc~ — insert the TOC at point.
- ~M-x org-skim-yank-toc~ — copy it to the kill ring instead.

Each heading is a clickable ~skim:~ link; follow with ~C-c C-o~ to jump to that page in Skim.

** Linking a page into Org (Skim → Org)

Capture a permanent link to the current page so you can jump back later.

- ~M-x org-skim-insert-page-link~ — insert a ~[[skim:PATH::PAGE][TITLE P.N]]~ link at point.
- ~M-x org-skim-yank-page-link~ — copy it to the kill ring.

Following the link (with ~C-c C-o~) opens the PDF in Skim and jumps to that page.

** Linking a Skim annotation to an Org heading (bidirectional)

Tie a Skim note to an Org heading by embedding a shared Org ID.  Once linked, you can jump in either direction.

From Skim (requires ~org-skim-note~):

- ~M-x org-skim-note-dwim~ — the main entry point.  If the selected note is already linked, display the Org entry narrowed to its subtree; otherwise start a capture.
- ~M-x org-skim-capture-note~ — force a new capture even if a link already exists.
- ~M-x org-skim-view-org-note~ — open the linked Org entry narrowed to its subtree.
- ~M-x org-skim-open-org-note~ — jump to the linked Org heading (no narrowing).

From Org (requires ~org-skim-note~):

- ~M-x org-skim-open-note-in-skim~ — open the PDF in Skim, navigate to the page, and select the note linked to the heading at point.
- ~M-x org-skim-open-note-in-skim-by-id~ — same, but resolved by an explicit Org ID (or the ~id:~ link / ~ID~ property at point).

** Controlling the reading bar

Move through text line-by-line without touching the mouse.

- ~M-x org-skim-toggle-reading-bar~ — show or hide the bar.
- ~M-x org-skim-reading-bar-next-line~ — advance one line (rolls to the next page at the bottom).
- ~M-x org-skim-reading-bar-previous-line~ — go back one line (rolls to the previous page at the top).

Bind these to keys for a comfortable read-along experience.

* Functions

** Table of contents

- ~M-x org-skim-insert-toc~ — insert the front Skim document's table of contents at point in the current buffer.  Each heading is a clickable ~skim:~ link.
- ~M-x org-skim-yank-toc~ — copy the table of contents to the kill ring instead.

The outline is rendered as a tree of Org headings whose depth is indicated by repeating ~org-skim-toc-header-char~.

** Page navigation

- ~M-x org-skim-next-page~ — advance the front Skim document by one page (no-op on the last page).
- ~M-x org-skim-previous-page~ — go back one page (no-op on the first page).
- ~M-x org-skim-goto-page~ — jump to a specific page.  A numeric prefix argument supplies the page (~C-u 12 M-x org-skim-goto-page~); with no prefix it prompts, defaulting to the current page.  Out-of-range values are clamped.  Lisp callers: ~(org-skim-goto-page 12)~.
- ~(org-skim-current-page)~ — non-interactive; returns the 1-based current page index.

** Page links

- ~M-x org-skim-insert-page-link~ — insert a ~[[skim:PATH::PAGE][DESC]]~ link at point.  ~DESC~ follows ~org-skim-page-link-format~.
- ~M-x org-skim-yank-page-link~ — copy the link to the kill ring.
- ~org-skim-open-page-link~ — registered as the ~skim:~ link handler; called by ~org-open-at-point~.

** Bookmarks

- ~M-x org-skim-bookmark-add~ — add a bookmark at the current page.  The name defaults to ~org-skim-bookmark-name-format~; a prefix argument (or ~org-skim-bookmark-prompt-name~ non-nil) prompts for a name prefilled from ~org-skim-bookmark-prompt-format~.  The container is chosen per ~org-skim-bookmark-container~.
- ~M-x org-skim-bookmark-remove~ — remove a bookmark chosen via completion.
- ~M-x org-skim-goto-bookmark~ — open a bookmark's PDF in Skim and jump to its page.
- ~(org-skim-bookmark-list)~ — non-interactive; returns every file bookmark as ~((folder name path page occurrence) ...)~, globals first, then sorted by folder, file basename, and name.

** Notes (requires ~org-skim-note~)

Interactive commands:

- ~M-x org-skim-note-dwim~ — the main entry point.  View the linked Org entry if it exists; otherwise start a capture.  A prefix argument forces the current frame.
- ~M-x org-skim-capture-note~ — capture an Org heading for the selected Skim note, embedding a shared Org ID.
- ~M-x org-skim-view-org-note~ — display the linked Org entry narrowed to its subtree.  Honours ~org-skim-view-note-in-new-frame~.
- ~M-x org-skim-open-org-note~ — jump to the linked Org heading via ~org-skim-open-org-note-function~.
- ~M-x org-skim-read-org-note~ — return the linked heading's body as a string without displaying it.
- ~M-x org-skim-open-note-in-skim~ — from Org, open the PDF in Skim and select the note linked to the heading at point.
- ~M-x org-skim-open-note-in-skim-by-id~ — same, resolved by an Org ID (from an ~id:~ link, ~ID~ property, or the minibuffer).
- ~M-x org-skim-set-bibtex-key~ — set or update the page-1 key-icon note that records the document's BibTeX key.

Non-interactive helpers:

- ~(org-skim-get-active-note)~ — returns a plist ~(:type :page :text :org-id :bounds)~ for the selected note, or nil.
- ~(org-skim-bibtex-key)~ — returns the BibTeX key embedded on page 1, or nil.
- ~(org-skim-get-org-id)~ — returns the selected note's embedded Org ID, or signals a ~user-error~.
- ~(org-skim-capture-value KEY)~ — for use in ~%(...)~ escapes in ~org-skim-capture-template~.

** Reading bar

- ~M-x org-skim-show-reading-bar~ — show the reading bar on the first line of the current page.  No-op if already visible.
- ~M-x org-skim-hide-reading-bar~ — hide the reading bar.
- ~M-x org-skim-toggle-reading-bar~ — flip between shown and hidden; when turning on, the bar lands on the first line of the current page.
- ~M-x org-skim-reading-bar-next-line~ — advance one line; rolls to the next page at the bottom, stops on the final line of the last page.
- ~M-x org-skim-reading-bar-previous-line~ — go back one line; rolls to the previous page at the top.

** Document

- ~M-x org-skim-save-document~ — save the front Skim document.

* Customization

| Variable                            | Default                       | Description                                                                                                                  |
|-------------------------------------+-------------------------------+------------------------------------------------------------------------------------------------------------------------------|
| ~org-skim-toc-header-char~          | ~?*~                          | Character repeated to build each heading level's prefix.                                                                     |
| ~org-skim-toc-header-name~          | ~"TOC"~                       | Root heading placed above the outline; items shift down one level below it. Set to ~nil~ for no root heading.                |
| ~org-skim-page-link-format~         | ~"${title} P${page}"~         | Template used to render the description of an inserted page link.                                                            |
| ~org-skim-bookmark-name-format~     | ~"P${page}"~                  | Template for the silent default name when adding a bookmark.                                                                 |
| ~org-skim-bookmark-prompt-format~   | ~"${title} P${page}"~         | Template used to prefill the prompt when ~org-skim-bookmark-add~ asks for a name.                                            |
| ~org-skim-bookmark-prompt-name~     | ~nil~                         | When non-nil, always prompt for the bookmark name on add.  A prefix argument to ~org-skim-bookmark-add~ flips this per call. |
| ~org-skim-bookmark-container~       | ~auto~                        | Where ~org-skim-bookmark-add~ places new bookmarks: ~auto~ prompts, ~nil~ adds at top level, or a folder name string.        |
| ~org-skim-open-org-note-function~   | ~org-id-find~                 | Function called with the Org ID to open a note from a Skim annotation.                                                       |
| ~org-skim-open-note-extensions~     | ~("pdf")~                     | File extensions ~org-skim-open-note-in-skim~ will open in Skim (lowercase, no leading dot).                                  |
| ~org-skim-note-file-function~       | ~org-skim--citar-note-file~   | Function mapping a citekey to its Org notes file (default resolves through citar's file-per-key layout).                     |
| ~org-skim-capture-template~         | (see docstring)               | ~org-capture~ template used by ~org-skim-capture-note~; splice note fields with ~%(org-skim-capture-value KEY)~.             |
| ~org-skim-capture-in-new-frame~     | ~nil~                         | When non-nil, run ~org-skim-capture-note~ in a dedicated frame, auto-deleted on finalize/abort (graphical Emacs only).      |
| ~org-skim-view-note-in-new-frame~   | ~nil~                         | When non-nil, ~org-skim-view-org-note~ (and the read branch of ~org-skim-note-dwim~) opens the note in a dedicated frame.   |
| ~org-skim-capture-frame-parameters~ | (see docstring)               | ~make-frame~ parameters for the dedicated capture and view frames (used when either ~*-in-new-frame~ option is non-nil).    |
| ~org-skim-note-title-template~      | ~"Notes on ${title}/${year}"~ | Template for the ~#+title~ of a newly created notes file (citar fields ~title~, ~author~, ~year~, ~citekey~).                |
| ~org-skim-note-filetags~            | ~"ReadingNote"~               | Tag(s) for the ~#+filetags~ of a new notes file; ~nil~ omits the line.                                                       |
| ~org-skim-note-references-property~ | ~"REFERENCES"~                | File-level property naming the ~@citekey~ reference in new notes files; ~nil~ omits it.                                      |
| ~org-skim-note-icon-size~           | ~16~                          | Size in points of the page-1 key-icon note created by ~org-skim-set-bibtex-key~.                                             |
| ~org-skim-applescript-backend~      | ~auto~                        | How AppleScript is executed: ~auto~ (prefer built-in), ~do-applescript~ (built-in OSA bridge), or ~osascript~ (subprocess).  |
| ~org-skim-debug~                    | ~nil~                         | When non-nil, force the ~osascript~ backend and log AppleScript error text to =*Messages*=.  See the Debugging section.      |

Template variables supported in ~org-skim-page-link-format~, ~org-skim-bookmark-name-format~, and ~org-skim-bookmark-prompt-format~: ~${title}~, ~${file}~, ~${path}~, ~${page}~, ~${date}~, ~${time}~.  Unknown variables are left in place.
